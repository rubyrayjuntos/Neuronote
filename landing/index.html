About RubyRayJuntos

RubyRayJuntos is a software innovation studio focused on building AI-first applications that prioritize safety, transparency, and user empowerment.

Our mission is to demonstrate that powerful AI capabilities and rigorous safety guarantees are not mutually exclusive. We believe the next generation of software will be malleable—adaptable at runtime by AI—and we're building the architecture to make that safe.

Flagship Product

Introducing NeuroNote

A dual-kernel architecture for safe, runtime-malleable software. AI generates declarative specifications—never executable code.

No Arbitrary Code Execution

AI produces JSON schemas (AppDefinitions), never JavaScript. The Host interprets through whitelisted components only.

Multi-Phase Verification

Every AI proposal passes through Zod schema validation, semantic checks, and embedded test vectors before acceptance.

Instant Rollback

JSON-Patch artifacts track every state change. Bad proposal? One-click rollback with full context preservation.

WASM Sandbox

Tier 1 operators execute in a QuickJS WebAssembly sandbox with strict resource budgets and message-passing isolation.

Transparent Diffs

Every migration generates RFC 6902 JSON-Patch artifacts. See exactly what the AI changed before accepting.

Lens Laws Verified

State access uses bidirectional lenses (LSI optics) with GetPut/PutGet laws verified at runtime.

The Dual-Kernel Architecture

A principled separation of concerns: the AI proposes, the Gatekeeper verifies, and the Host executes through trusted, sandboxed pathways.

Academic Research

// Paper Abstract

Malleable Software Architecture for Embedded AI Safety

We present a dual-kernel architecture for building software that can be safely modified at runtime by AI agents. Our approach treats AI-generated artifacts as untrusted input, requiring multi-phase verification before execution. We demonstrate that bidirectional lenses (LSI optics) combined with immutable state management (Immer) provide mathematical guarantees for state migration, while WASM sandboxing ensures computational isolation. Our implementation achieves 364 passing tests with zero arbitrary code execution vulnerabilities.
